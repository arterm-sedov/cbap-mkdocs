### Улучшенный глобальный поиск

Добавлена поддержка расширенного глобального поиска по документам и вложениям:

- Поиск по свойствам документа: теперь можно выполнять поиск по текстовым полям внутри документов.
- Поиск по именам вложений: поддерживается поиск по названию вложенных файлов.
- Поиск по содержимому вложений: реализована возможность поиска по текстовому содержимому документов форматов "txt", "csv", "xml", "eml", "mhtml", "htm", "html", "docx", "xlsx".
- Работа с multivalue-документами и ревизиями: поиск выполняется корректно, включая возможность поиска по ревизиям. Если строка найдена в ревизии, отображается оригинальное имя файла этой ревизии.
- Документы доступны для поиска: атрибут типа "Документ" теперь можно пометить как доступный для поиска, что позволяет выполнять поиск по содержимому документов.
- Обновление данных поиска: после загрузки нового документа, новой ревизии или обновления атрибутов объекта, поиск корректно работает по старым и новым данным, не перезаписывая их.

### Расширенная фильтрация (фасетный поиск)

В Comindware Business Application Platform добавлена возможность использования фасетного поиска, что позволяет пользователям и аналитикам гибко настраивать формы поиска и фильтрации по атрибутам записей, коллекций и таблиц. Новая функциональность дает возможность выполнять поиск по более сложным критериям, а также использовать формы поиска для работы с атрибутами типа "Запись".

#### Основные возможности

1. **Формы фасетного поиска**:
   - Добавлен новый тип форм — "Форма поиска", который можно настроить для таблиц, коллекций и атрибутов типа "Запись".
   - Формы поиска позволяют пользователям осуществлять расширенный поиск по атрибутам записи.
   - Поддерживаются различные типы атрибутов, такие как гиперссылки, связанные процессы и чертежи, с возможностью указания логических операторов поиска.

2. **Фильтрация данных**:
   - Формы поиска работают с полнотекстовым поиском по индексированным текстовым атрибутам.
   - При поиске в таблицах и коллекциях фильтры могут применяться как на уровне колонок, так и через формы поиска.
   - Поддерживается фильтрация записей как по отображаемым данным, так и по всем записям, связанным с шаблоном.

3. **Поиск по атрибутам типа "Запись"**:
   - В атрибуте типа "Запись" добавлена возможность расширенного поиска, который открывается через диалоговое окно. Пользователи могут выбирать значения из таблиц и добавлять их в поле атрибута.
   - Поддерживается множественный выбор записей, при этом отображаются только те записи, которые еще не были выбраны.

#### Улучшенные элементы интерфейса

1. **Виджеты фильтров**:
   - Виджеты фильтров позволяют выбирать операторы поиска в зависимости от типа атрибута. Для каждого атрибута автоматически предлагается соответствующая операция фильтрации.
   - В "расширенном" режиме фильтра пользователи могут выбирать дополнительные операции поиска для более точной фильтрации данных.

2. **Контекстное меню чекбоксов в таблицах**:
   - Добавлена возможность выбора всех записей либо только отображаемых записей в таблице с учетом настроенных фильтров.

#### Сценарии использования

- **Расширенный поиск в таблицах и коллекциях**: пользователи могут настраивать фильтры по атрибутам и колонкам для точной выборки данных.
- **Поиск по атрибуту типа "Запись"**: при множественном выборе записей добавлена возможность фильтрации и отображения только выбранных записей через специальную вкладку.

#### Ограничения

1. Невозможно переключить тип формы между "Публичной", "Внутренней" и "Формой поиска" без создания новой формы.
2. В форме поиска не поддерживается статичный текст, кнопки, вкладки и группы вкладок.

### Улучшения в работе с диаграммами

#### Описание

Для повышения удобства создания диаграмм и улучшения визуальной четкости была добавлена возможность автоматического выравнивания элементов, а также ручного регулирования потоков. Эти изменения помогут пользователям более точно размещать элементы и настраивать направления потоков на диаграмме, обеспечивая аккуратный и организованный вид диаграмм.

#### Основные возможности

1. **Автоматическое выравнивание элементов**:
   - При перемещении элементов на диаграмме система автоматически сравнивает линии границ и центров элемента с другими элементами. Если элементы выровнены по линиям границ или центров, система отображает линии выравнивания. Это позволяет легко упорядочивать элементы на диаграмме.
   - Линии выравнивания появляются при соприкосновении элементов и исчезают при дальнейшем движении.

2. **Ручное регулирование стрелок потоков**:
   - Стрелки потоков теперь имеют контрольные точки на каждом изгибе, которые позволяют вручную корректировать направление потока.
   - В случае прямых стрелок добавлены центральные точки для выравнивания и регулирования каждой половины потока.
   - При добавлении изгибов появляются новые точки для регулирования, что позволяет гибко настраивать путь стрелки.

3. **Указание потоков на элементы**:
   - Теперь поток можно направить не только на конкретный коннектор, но и на сам элемент. Система автоматически определяет ближайший коннектор для корректного отображения потока.

4. **Коннекторы по сетке**:
   - Коннекторы элементов теперь выравниваются по сетке диаграммы, что исключает искривление стрелок потоков и обеспечивает их симметричное отображение.

5. **Встраивание элементов в потоки**:
   - Пользователи могут встроить элемент прямо в поток. В таком случае поток автоматически разделяется на две части: одна часть становится входящей в элемент, а другая — исходящей.

#### Преимущества

Эти улучшения значительно повышают удобство работы с диаграммами, позволяют быстрее и точнее строить диаграммы, а также обеспечивают эстетичный и аккуратный вид конечного результата.
